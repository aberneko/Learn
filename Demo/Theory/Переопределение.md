## Определение и Цель
|    Аспект    | Описание                                                                                                                                                         | 
|:------------:|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------|
|   Что это?   | Механизм ООП, при котором дочерний класс предоставляет свою специфичную реализацию для метода, который уже был объявлен в его родительском классе.               | 
| Главная Цель | Реализация полиморфизма (один интерфейс, множественные формы поведения). Позволяет объектам разных классов реагировать по-разному на один и тот же вызов метода. | 
| Терминология | Метод в родительском классе называется переопределяемым (Overridden Method), метод в дочернем классе — переопределяющим (Overriding Method).                     |  

## Правила (Сигнатура и Доступ)

#### _Чтобы метод считался переопределенным, он должен строго соответствовать следующим правилам:_

|                         |                                                                                                                                                          |
|:-----------------------:|:---------------------------------------------------------------------------------------------------------------------------------------------------------|
|      **Сигнатура**      | Должна быть идентичной (то же имя, те же типы и количество аргументов).                                                                                  |
|  **Возвращаемый Тип**   | Должен быть тем же самым, либо ковариантным (подклассом оригинального возвращаемого типа — в Java это разрешено с JDK 1.5).                              |
| **Модификатор Доступа** | Не может быть более ограничивающим (например, нельзя переопределить `public` метод как `private` или `protected`).                                       |
|     **Исключения**      | Переопределяющий метод не может объявлять новые, более широкие проверяемые исключения (Checked Exceptions), чем те, что объявлены в родительском методе. |
|    **Наследование**     | Происходит только между классами, связанными отношением наследования (суперкласс и подкласс).                                                            |

## Ключевые слова и Инструменты Java

| Инструмент  | Назначение                                                                                                                                                                              |
|:-----------:|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `@Override` | **Аннотация.** Рекомендуется использовать. Помогает компилятору проверить, что вы правильно соблюли все правила переопределения. Если сигнатура не совпадает, компилятор выдаст ошибку. |
|   `super`   | **Ключевое слово.** Позволяет вызвать реализацию родительского (суперкласса) метода внутри переопределяющего метода.                                                                    |
|   `final`   | **Модификатор.** Если метод в суперклассе помечен как final, его нельзя переопределить в подклассах.                                                                                    |
|  `static`   | **Модификатор.** Статические методы нельзя переопределять, только скрывать `(Hiding)`.                                                                                                  |
|  `private`  | **Модификатор.** Приватные методы не наследуются и, следовательно, не могут быть переопределены.                                                                                        |

## Связь с Полиморфизмом и Upcasting'ом

#### _Переопределение — это механизм, который делает возможным динамический полиморфизм (Позднее связывание):_
|    Концепция     | Объяснение                                                                                                                                                            | 
|:----------------:|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|   `Upcasting`    | Создание ссылки типа суперкласса для объекта подкласса. Пример: `Shape s = new Circle()`;                                                                               | 
|  **Компилятор**  | На этапе компиляции проверяет тип ссылки (Shape), чтобы убедиться, что метод вообще существует и доступен.                                                            | 
| `JVM (Run Time)` | На этапе выполнения использует динамическую диспетчеризацию (смотрит на фактический тип объекта, `Circle`), чтобы запустить правильную, переопределенную версию метода. |
