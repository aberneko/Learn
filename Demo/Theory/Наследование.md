# Наследование  
> ### Позволяет создавать новые классы на основе уже существующих
> + #### Благодаря наследованию дочерние классы автоматически приобретают функционал класса-родителя. Не нужно раз за разом прописывать одни и те же поля и методы — их можно передать по наследству.
---

> ### Для передачи полей одного класса к другому требуется ключевое слово `extends`
> ```
> public class Animal {}
> public class Dog extends Animal {}
> ```
> 
> + #### Важно запомнить, что у классов в Java может быть сколько угодно предков, но только один родитель. То есть при помощи ключевого слова `extends` можно наследовать только от одного класса. Написать `class Fox extends Animal, Canidae` не получится.
> + #### При внесении изменений в родительский класс они автоматически перейдут по наследству.
> ```
> public class Practicum {
>    public static void main(String[] args) {
>        Ellipse Ellipse = new Ellipse();
>        Ellipse.drawCircle();
>        Ellipse.drawSquare();
>        Ellipse.drawRectangle();
>        Ellipse.drawCircle();
>        Ellipse.circleToEllipse();
>    }
>}
>
> class Figure {
>   public void drawCircle() {
>       System.out.println("Рисуем круг");
>   }
>    public void drawSquare() {
>        System.out.println("Рисуем квадрат");
>    }
>    public void drawRectangle() {
>        System.out.println("Рисуем прямоугольник");
>    }
> }
>
> class Circle extends  Figure{
> }
>
> class Square   extends Circle {
> }
>
> class Rectangle extends Square {
> }
>
> class Ellipse extends Square{
>   public void circleToEllipse() {
>       System.out.println("Превращаем круг в овал");
>   }
> }
> ```
 ---
> #### Класс-родитель, или класс-предок, иначе ещё называют `суперклассом`. Классы-потомки — `подклассами`.
> #### `Подкласс` наследует все поля и методы `суперкласса`, но прямой доступ имеет только к тем, у которых область видимости `public, protected или package-private` (если класс-потомок находится в одном пакете с классом-родителем). Доступ к полям и методам, помеченным модификатором `private`, при расширении с помощью extends, подкласс не имеет.
> #### Можно `переопределить методы`: реализовать в подклассе методы, которые будут иметь ту же сигнатуру, что и в `суперклассе`, но отличаться своим поведением.
>```
>class Coffee {
>   protected double espresso;
>   protected double milk;
>
>    public Coffee() {
>        espresso = 50.0;
>        milk = 250.0;
>    }
>
>   public double mixAndGetVolume() {
>       return espresso + milk;
>   }
>}
>
>class Cappuccino extends Coffee {
>
>   public double mixAndGetVolume() { // переопределили метод mixAndGetVolume()
>       double milkFoam = milk / 2;
>       return espresso + milkFoam + milk / 3;
>   }
>
>}
>
>public class Practicum {
>   public static void main(String[] args) {
>   Cappuccino cappuccino = new Cappuccino();
>   System.out.println("Объём капучино - " + cappuccino.mixAndGetVolume());
>   }
>} 
>```
